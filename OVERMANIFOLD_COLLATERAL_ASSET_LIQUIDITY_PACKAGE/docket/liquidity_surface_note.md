# Liquidity Surface Note - Overmanifold

## Primitive

`Ω_link = (H, W, P, C, R, S, K, L)`

Where:

- H = handlers: SMS, Telegram, email, Signal, Matrix
- W = wallets: Solana, EVM, Bitcoin, Lightning, stablecoin rails
- P = proofs: signatures, Merkle roots, artifact hashes, repo proofs
- C = capabilities: receive, send, relay, compute, infer, verify
- R = routing surfaces: web, SMS fragments, QR, Bluetooth, IPFS, relays
- S = settlement mappings: programs, escrows, relayers, sponsor vaults
- K = KPI vector: uptime, trust, novelty, proof quality, route success
- L = liquidity state: reachable verified economic capacity

## Doctrine

The link is the liquidity pool.

A normal link is passive. An Overmanifold link is active: it can resolve identity, negotiate capability, route intent, carry proof, sponsor settlement, and update appraisal state.

## Boundary

This package proves integrity and provenance of files through hashing. It does not prove intrinsic dollar value. Dollar value emerges only from market acceptance, licensing, financing, revenue, or protocol choice.
